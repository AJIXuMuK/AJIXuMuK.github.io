--- layout: post title: Performace of querying items in lists and libraries date: '2015-06-18T16:06:00.000-07:00' author: Alex Terentiev tags: - SharePoint Online - CamlQuery - List.GetItems - SharePoint 2010 - C# - SP.CamlQuery - SharePoint - JavaScript - GetItems - CAML - SharePoint 2013 - Client Side Object Model - CSOM - SP.List.GetItems modified_time: '2015-06-18T16:06:58.117-07:00' blogger_id: tag:blogger.com,1999:blog-3066084330774405472.post-6857954473712572101 blogger_orig_url: http://blog.aterentiev.com/2015/06/performace-of-querying-items-in-lists.html --- Some time ago I was working on project where I needed to query items from several lists and document libraries (each list and library contained 200+ items). I needed to display Title and Url fields from lists and file name and file url from document libraries.
My first code worked too long and I've started to think about increasing the performance of queries.
And here are the things that helped me to increase speed in several times:

  1. Use <ViewFields> inside query instead of Include(...) inside ClientContext.Load()
  2. Use FileRef and FileLeafRef fields instead of File field (which is an object with several properties in it).

That's all. These two changes helped me a lot. And the customer is happy about the performance they have.
